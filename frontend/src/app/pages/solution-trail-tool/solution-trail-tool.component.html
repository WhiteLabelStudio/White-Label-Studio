What type of uplink role configuration is needed?
Dual Uplink Category Test Cases
  • Validate that when both uplinks are tagged as "primary" traffic is subject to per flow ECMP over both uplinks. This applies for traffic not configured with an AAR SLA.
  • Validate that when one uplink is tagged as "primary" and the other uplink as "secondary" traffic always takes the primary uplink.
  • Validate that when one uplink is tagged as "primary" and the other uplink as "secondary" and the "primary" uplink fails that all traffic is sent over the secondary uplink..
  • Validate that an advanced forwarding policy can be created to send HTTP traffic over "secondary" uplink only.
  • Validate that an advanced forwarding policy can be created to send HTTP traffic over "secondary" uplink and if it fails then over the primary uplink.
